# Design Thinking

Human-centered design through the complete design thinking methodology. Creates solutions deeply rooted in user needs by combining empathy-driven research with systematic creative problem-solving.

## The Five Phases

### 1. Empathize
Build deep understanding of users through research and observation.

**Methods from library:**
- User interviews
- Observation sessions
- Empathy mapping
- Journey mapping
- Shadowing

### 2. Define
Transform insights into clear problem statements.

**Outputs:**
- Point of View statement: "[User type] needs [need] because [insight]"
- "How Might We" questions
- Problem framing

### 3. Ideate
Generate diverse solutions through structured brainstorming.

**Methods:**
- Brainstorming (classic)
- Brainwriting
- SCAMPER
- Mind mapping
- Worst possible idea

### 4. Prototype
Make ideas tangible quickly.

**Formats:**
- Sketches
- Wireframes
- Mockups
- Physical prototypes
- Role-play

### 5. Test
Validate with real users.

**Approaches:**
- Usability testing
- A/B testing
- Feedback sessions
- Field testing

## Usage

When working on a design challenge:

1. **Load this skill** for design thinking facilitation
2. **Start with empathy** - understand users before solving
3. **Frame the problem** - define clearly before ideating
4. **Generate widely** - defer judgment, aim for quantity
5. **Prototype quickly** - make ideas tangible
6. **Test early** - learn from real users
